# AGENTS.md

## 목적
이 저장소의 Git/PR 워크플로우에 대한 단일 진실 공급원(SSoT) 문서다.

## 워크플로우 정책 (엄격)
요약: 모든 변경은 작업 브랜치와 `main` 대상 PR을 통해서만 반영한다.

- `main`에서 직접 작업하지 않는다.
- 모든 작업은 새 브랜치에서 시작한다.
- 커밋/푸시는 작업 브랜치에서만 수행한다.
- PR 대상 브랜치는 반드시 `main`으로 한다.
- 반영은 PR 머지로만 진행한다.

## 브랜치 생명주기
요약: 모든 작업은 동일한 브랜치 절차를 따른다.

1. 로컬 `main`을 최신 상태로 갱신한다.
2. 작업 브랜치를 생성하고 전환한다 (기본: `feature/<topic>`).
3. 변경 작업 후 명확한 메시지로 커밋한다.
4. 작업 브랜치를 원격에 푸시한다.
5. `main` 대상 PR을 생성하고 리뷰/체크를 완료한다.
6. PR을 머지한다.
7. 머지된 작업 브랜치를 로컬/원격에서 모두 삭제한다.
8. `main`으로 복귀하고 최신 상태를 동기화한다.

## PR 작성 베스트 프랙티스
요약: PR은 집중도, 검증 가능성, 리뷰 용이성을 기준으로 작성한다.

- 제목은 간결하고 행동 중심으로 작성한다.
- 제목/본문 기본 언어는 한글로 작성한다. 기술 용어/고유명사는 필요 시 영어를 사용한다.
- PR 하나에는 하나의 목적만 담고, 혼합 변경을 피한다.
- 본문에는 다음 항목을 반드시 포함한다:
  - 변경 배경(왜 필요한가)
  - 주요 변경 사항(핵심만)
  - 테스트 방법/결과(실행 명령 포함)
  - 리스크 및 롤백 방법
- 리뷰 가능한 크기로 유지하고, 큰 변경은 분리한다.
- 머지 전 리베이스/동기화로 통합 리스크를 낮춘다.

## 학습/기술블로그 문서 규칙
요약: 공부한 내용은 재사용 가능한 기술블로그형 Markdown으로 남긴다.

- 학습 결과는 `study/` 하위에 Markdown 문서로 작성한다.
- 파일명은 반드시 `yyyy-MM-dd_${title}.md` 형식을 따른다.
- 기존 문서가 형식을 따르지 않으면 우선 정규화(리네임)한다.
- 문서는 요약, 사용법, 동작 방식, 응용, 유의사항을 포함하도록 우선 구성한다.

## CLI 도구 활용 규칙
요약: 대체 가능한 경우 더 효율적인 로컬 CLI 도구를 우선 사용한다.

- 사용 가능 도구: `git`, `gh`, `jq`, `fzf`, `zoxide`, `fd`, `direnv`, `nvm`, `openjdk`, `grpcurl`, `httpie`, `watch`, `telnet`, `tmux`, `ripgrep`
- 동일 목적이라면 기존 습관적 명령보다 위 도구를 우선 검토한다.
- 검색/파일 탐색은 `ripgrep(rg)`, `fd`를 우선 사용한다.
- GitHub 연동 작업은 `gh`를 기본 도구로 사용한다.

## SSoT 운영 원칙
요약: 운영 규칙의 중복과 충돌을 허용하지 않는다.

- Git/PR 운영 정책은 이 파일에만 정의한다.
- 학습/문서 작성 규칙도 이 파일에만 정의한다.
- 다른 문서에 동일 규칙을 중복 작성하지 않는다.
- 다른 문서는 `AGENTS.md`를 참조만 하고 정책 상세를 재서술하지 않는다.
- 다른 문서와 충돌 시 `AGENTS.md`를 우선한다.
- 정책 변경은 반드시 이 파일을 먼저 수정한다.
